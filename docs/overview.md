# Overview

## What is LifeSimulation?

LifeSimulation ("The Sim") is a text-based life simulation game that runs entirely in
the browser as a Progressive Web App (PWA). It is structurally similar to games like
BitLife, but it is narrative-heavy: instead of canned event text, the moment-to-moment
prose is generated by large language models (LLMs), while all mechanics are resolved in
deterministic JavaScript.

The core design principle is a strict division of labor:

> The AI handles narration only. Everything else — math, time, risk, routing — lives in
> code.

This keeps the simulation consistent and reproducible, avoids the AI "self-reporting"
stats, and dramatically reduces the number of tokens sent to the models each turn.

## What it does

- **Simulates a life over many turns.** The player types actions in natural language.
  Each turn advances in-game time, applies stat changes (health, energy, hunger, mood,
  cash, and more), rolls for risks (illness, accidents, death), and records history.
- **Manages a persistent world.** A single `world_state` object is the source of truth,
  persisted locally in the browser via IndexedDB (Dexie.js) and exportable as JSON for
  save/load/backup.
- **Models NPCs.** Significant NPCs have numeric trait arrays (warmth, jealousy,
  patience, honesty, etc.), relationship and trust meters, weekly schedules, and status
  flags. Their behavior and prominence in the UI evolve based on interactions.
- **Generates narration on demand.** Notable, crisis, and death turns are narrated by a
  creative model; routine turns are narrated cheaply or skipped; novel actions are
  classified into mechanical effects before narration.
- **Runs anywhere without a build step.** It is a static site — open `sim/index.html` or
  serve the `sim/` folder.

## The three-path input model

Every player input is intercepted by a JavaScript sanitizer before any API call, and
routed down one of three paths:

1. **Explicit input** — detected in code, classified with a hardcoded activity table,
   then narrated by the creative model. The classifier LLM never sees explicit content.
2. **Novel non-explicit input** — classified by a JSON-only model (action type, time
   cost, stat deltas, risk class); code applies the deltas, then the creative model
   narrates.
3. **Routine autopilot** — mundane time-skips (sleep, commute, standard shift) are
   narrated directly by the cheaper model from a sanitized world state, with no creative
   model call.

## Turn classification

After the engine resolves a turn, it is classified using explicit constants (not
"vibes") into one of four tiers, which controls how much narration is produced:

- `ROUTINE`
- `NOTABLE`
- `CRISIS`
- `DEATH`

## AI providers

The app is provider-agnostic. It ships with a universal dispatch layer
(`sim/providers.js`) that auto-detects the provider from the API key format and routes
requests to the correct endpoint and auth format. Supported providers include Grok/xAI,
OpenAI, Anthropic, OpenRouter, Groq, and Google Gemini, with a fallback chain for
outages and refusals. All API keys are stored locally on the device.
